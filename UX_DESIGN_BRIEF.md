# 랜덤 자리 뽑기 디자이너 전달용 UX 브리프

## 프로젝트 개요
이 프로젝트는 교실, 스터디, 모임에서 사용할 수 있는 `로컬 기반 랜덤 자리 뽑기 웹앱`입니다.  
핵심은 단순한 추첨 도구가 아니라, 실제 좌석 구조와 운영 흐름을 반영한 `자리 배정 도구`로 보이게 만드는 것입니다.

현재 기능은 충분한 편이며, 이번 디자인 작업의 목적은 `기능 추가`가 아니라 `기능 노출 순서와 사용 흐름을 더 쉽게 만드는 것`입니다.

## 제품 목표
- 처음 사용하는 사람도 10초 안에 흐름을 이해할 수 있어야 함
- 기본 사용자는 복잡한 옵션 없이 바로 자리표를 만들 수 있어야 함
- 고급 사용자는 통로, 고정석, 일부 재추첨 같은 기능을 필요할 때만 꺼내 쓸 수 있어야 함
- 결과 자리표가 항상 화면에서 가장 중요한 영역처럼 보여야 함

## 주요 사용자
### 1. 기본 사용자
- 교사, 운영자, 진행자
- 목표: 학생 명단만 넣고 바로 자리 뽑기
- 특징: 복잡한 설정을 싫어함

### 2. 고급 사용자
- 실제 교실 구조를 반영하려는 사용자
- 목표: 통로/비활성 좌석, 고정석, 공정성 옵션까지 활용
- 특징: 기능은 필요하지만 항상 보일 필요는 없음

### 3. 반복 사용자
- 같은 반/팀으로 여러 번 사용하는 사용자
- 목표: 템플릿 불러오기, 이력 다시 보기, 일부만 재추첨
- 특징: 운영 기능이 중요하지만 기본 화면을 복잡하게 만들고 싶지는 않음

## 핵심 UX 원칙
- `처음엔 단순하게, 필요할 때만 고급 기능을 드러낸다`
- `입력 -> 좌석 -> 자리 뽑기 -> 결과` 흐름이 한눈에 보여야 한다
- 결과판은 언제나 메인 콘텐츠처럼 보여야 한다
- 템플릿과 이력은 중요하지만 본문이 아니라 보조 채널에서 열어야 한다

## 권장 정보 구조
### 상단 헤더
- 앱 이름
- 현재 상태 요약
- `템플릿` 버튼
- `이력` 버튼

### 좌측 기본 플로우 패널
#### 1. 명단 입력
- 이름 입력
- 인원 수 표시
- 파싱 미리보기

#### 2. 좌석 설정
- 행/열 입력
- 추천 좌석 배치 3개
- 사용 가능 좌석 수 / 남는 좌석 수

#### 3. 실행
- 메인 CTA: `자리 뽑기`
- 상태 메시지 / 오류 메시지

### 좌측 하단 고급 설정
- 기본은 접힘
- 내부 항목
  - 지난 자리 피하기
  - 자리 편중 줄이기
  - 고정석 지정
  - 좌석 직접 편집

### 우측 결과판
- 결과판 헤더
  - 이름 검색
  - 복사
  - 인쇄
- 자리표 본문
- 결과 후 액션
  - 전체 다시 뽑기
  - 일부만 다시 뽑기
- 일부 재추첨 대상 선택 UI는 버튼 클릭 후에만 노출

## 꼭 지켜야 하는 노출 규칙
- 첫 진입 시 보여야 하는 것
  - 명단 입력
  - 좌석 설정
  - 자리 뽑기 버튼
  - 결과 빈 상태
- 기본 화면에서 숨겨야 하는 것
  - 템플릿 목록
  - 이력 목록
  - 좌석 직접 편집
  - 고정석 상세 UI
  - 공정성 옵션 상세
  - 일부 재추첨 대상 선택 UI
- 결과가 생긴 뒤에만 보여야 하는 것
  - 전체 다시 뽑기
  - 일부만 다시 뽑기
  - 일부 재추첨 대상 선택 UI

## 핵심 유스케이스
### 유스케이스 1. 빠르게 자리 뽑기
- 명단 입력
- 좌석 수 확인
- 자리 뽑기
- 결과 확인

### 유스케이스 2. 실제 교실 구조로 배정
- 고급 설정 열기
- 통로/비활성 좌석 편집
- 자리 뽑기

### 유스케이스 3. 특정 학생 고정
- 고급 설정 열기
- 학생과 좌석 선택
- 고정석 지정 후 자리 뽑기

### 유스케이스 4. 저장된 템플릿 사용
- 상단 템플릿 버튼
- 템플릿 불러오기
- 바로 자리 뽑기

### 유스케이스 5. 일부 학생만 다시 뽑기
- 첫 결과 생성
- `일부만 다시 뽑기` 선택
- 학생 선택
- 다시 배정

## 카피 가이드
- `랜덤 배정` 대신 `자리 뽑기`
- `선택 학생만 재추첨` 대신 `일부만 다시 뽑기`
- `직전 1회 동일 좌석 회피` 대신 `지난 자리 피하기`
- `균형 배정` 대신 `자리 편중 줄이기`
- 용어는 설정 중심보다 행동 중심으로 보여야 함

## 시각적 우선순위
### 가장 강조
- 결과 자리표
- 메인 CTA 버튼

### 중간 강조
- 명단 입력
- 좌석 설정

### 낮은 강조
- 템플릿 / 이력
- 고급 설정
- 보조 옵션

## 인터랙션 가이드
- 고급 설정은 `아코디언`
- 템플릿/이력은 `드로어`
- 결과 후 액션은 `인라인 버튼`
- 일부만 다시 뽑기는 버튼 클릭 후에만 학생 선택 UI 노출
- 검색은 결과판 안에서만 동작

## 반응형 방향
- 데스크톱: 좌측 플로우 + 우측 결과 2열
- 태블릿: 2열 유지 가능하되 밀도 축소
- 모바일: 세로 스택 구조
- 모바일에서도 결과판이 아래로 밀리더라도 구조 이해는 유지되어야 함

## 디자이너가 특히 봐야 하는 포인트
- 기본 사용자에게 과하지 않게 보이는가
- 결과판이 충분히 중심적인가
- 고급 기능이 숨겨져 있어도 존재를 인지할 수 있는가
- 템플릿/이력 드로어가 메인 흐름을 방해하지 않는가
- 결과 이후 액션이 자연스럽게 이어지는가
