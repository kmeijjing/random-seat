# 랜덤 자리 뽑기 확장 로드맵

## Summary
- 현재 앱을 `단순 랜덤 배정기`에서 `교실 운영용 자리 배정 도구`로 확장한다.
- 구현은 `1단계 실제 배치`, `2단계 운영 편의`, `3단계 공정성 강화` 순서로 나눈다.
- 모든 데이터는 계속 `localStorage`에 저장하고 서버는 두지 않는다.
- 각 단계는 독립 배포 가능해야 하며, 앞 단계 결과를 기반으로 다음 단계가 확장된다.

## Important Interfaces
- `SeatConfig`는 `rows`, `columns` 외에 `layout`, `recommendedLayouts`를 포함한다.
- 새 타입을 추가한다: `SeatCell`, `SeatCellType`, `FixedSeat`, `SeatTemplate`, `DrawHistoryEntry`, `DrawOptions`, `BalanceStats`.
- `SeatCellType`은 최소 `seat`, `aisle`, `blocked`를 지원한다.
- `SeatAssignment`는 `cellId`, `isFixed`, `zone`을 포함하도록 확장한다.
- `SavedState`는 `currentDraft`, `templates`, `history`, `searchQuery`, `selectedParticipantsForRedraw`를 포함하는 구조로 바꾼다.
- 배정 엔진은 `generateAssignments(layout, participants, options, history)` 형태로 확장한다.
- 템플릿은 `명단 + 좌석배치 + 고정석 + 통로 설정` 전체를 저장한다.
- 이력은 최근 20회만 저장한다.

## Phase 1: 실제 교실 배치 지원
### 목표
- 실제 교실처럼 `빈칸/통로`와 `고정석`이 있는 자리 배정을 지원한다.
- 학생 수에 맞는 좌석판 추천과 결과 검색을 제공한다.

### 구현 변경
- 좌석판을 단순 `rows x columns` 계산이 아닌 `셀 기반 레이아웃`으로 바꾼다.
- 각 셀은 `좌석`, `통로`, `비활성`으로 토글 가능하게 한다.
- `학생 수 자동 맞춤`은 자동 적용이 아니라 `추천 행/열 조합 3개 제안`으로 구현한다.
- 추천 기준은 `학생 수 수용`, `빈좌석 최소화`, `비율 과도함 방지`다.
- `고정석`은 명단에서 학생을 선택해 특정 좌석 셀에 고정하는 방식으로 구현한다.
- 고정석 학생은 랜덤 배정 풀에서 제외한다.
- 검색은 결과판 상단에서 학생 이름을 찾고 좌석을 하이라이트하는 방식으로 구현한다.
- 배정 로직은 `고정석 배치 -> 사용 가능 좌석 계산 -> 나머지 학생 셔플` 순서로 처리한다.

## Phase 2: 반복 운영 기능
### 목표
- 반/팀 단위로 명단과 배치를 반복해서 사용할 수 있게 한다.
- 결과를 보관하고 공유하기 쉽게 만든다.

### 구현 변경
- 템플릿 저장은 `명단 + 좌석배치 + 고정석 + 통로 설정` 전체를 저장한다.
- 템플릿 패널에서 `저장`, `불러오기`, `이름 변경`, `삭제`를 지원한다.
- 이력은 추첨 완료 시 자동 저장하고 최근 20회만 유지한다.
- 이력 항목은 `시각`, `참여자 수`, `좌석 수`, `사용 옵션`을 표시한다.
- 복사는 현재 자리표를 `행-열: 이름` 텍스트 형식으로 클립보드에 복사한다.
- 인쇄는 별도 페이지 없이 현재 결과판 중심의 인쇄용 CSS를 제공한다.

## Phase 3: 공정성 기반 배정
### 목표
- 단순 랜덤이 아니라 반복 사용 시 편중이 줄어드는 배정으로 확장한다.
- 전체 재추첨과 부분 재추첨을 지원한다.

### 구현 변경
- `재추첨 옵션`은 `전체 재추첨`과 `선택 학생만 재추첨` 두 가지를 지원한다.
- 선택 학생 재추첨은 나머지 학생과 고정석을 유지한다.
- `이전 자리와 겹치지 않게`는 `직전 1회 동일 좌석 회피`를 기본으로 구현한다.
- 동일 좌석 회피가 불가능하면 예외 허용으로 배정은 계속 진행한다.
- `균형 배정`은 `앞/뒤 + 좌/우` 기준으로 최근 이력 편중을 줄이는 점수 방식으로 구현한다.
- 좌석은 `front/back`, `left/right` 구역 태그를 가진다.
- 배정 엔진은 `금지 좌석 제외 -> 후보 좌석 계산 -> 균형 점수 부여 -> 낮은 패널티 우선 배정` 구조로 확장한다.
- 알고리즘은 완전탐색이 아니라 휴리스틱 기반으로 유지한다.
