# 랜덤 자리 뽑기 유스케이스

이 문서는 현재 앱의 핵심 사용자 흐름을 정리한 문서입니다.  
목적은 `무엇을 만들었는가`보다 `사용자가 어떤 상황에서 어떤 순서로 기능을 쓰는가`를 명확히 하는 데 있습니다.

## 사용자 유형
- `기본 사용자`: 빠르게 자리만 뽑고 싶은 교사/운영자
- `고급 사용자`: 실제 교실 배치, 고정석, 일부 재추첨까지 쓰는 사용자
- `반복 사용자`: 템플릿과 이력을 반복 활용하는 사용자

## 유스케이스 1. 처음 들어와서 빠르게 자리 뽑기
### 목표
- 최소한의 입력만으로 빠르게 자리표를 만든다.

### 시작 상태
- 앱 첫 진입
- 저장된 결과가 없거나 새 초안 상태

### 사용자 행동
1. 명단 입력 영역에 학생 이름을 붙여넣는다.
2. 좌석 설정에서 행/열을 확인하거나 추천 배치를 선택한다.
3. `자리 뽑기` 버튼을 누른다.

### 시스템 반응
- 명단을 자동 파싱해 인원 수와 미리보기를 보여준다.
- 사용 가능한 좌석 수를 계산한다.
- 랜덤 배정을 실행하고 결과판에 자리표를 표시한다.

### 완료 조건
- 모든 학생이 배정된 자리표가 우측 결과판에 표시된다.

## 유스케이스 2. 실제 교실 배치로 자리 뽑기
### 목표
- 통로와 비활성 좌석이 포함된 실제 교실 구조에 맞춰 배정한다.

### 시작 상태
- 명단 입력 완료
- 기본 좌석판이 생성된 상태

### 사용자 행동
1. `고급 설정`을 연다.
2. `좌석 직접 편집`을 펼친다.
3. 각 셀을 눌러 좌석, 통로, 비활성 상태로 바꾼다.
4. `자리 뽑기`를 실행한다.

### 시스템 반응
- 실제 `seat` 타입 셀만 배정 가능 좌석으로 계산한다.
- 통로와 비활성 셀은 결과판에서 배정 제외 상태로 보인다.

### 완료 조건
- 교실 배치 구조를 반영한 자리표가 생성된다.

## 유스케이스 3. 특정 학생을 고정석으로 배정하기
### 목표
- 특정 학생을 지정 좌석에 반드시 앉히고 나머지만 랜덤 배정한다.

### 시작 상태
- 명단 입력 완료
- 사용 가능한 좌석이 존재

### 사용자 행동
1. `고급 설정`을 연다.
2. `고정석 지정`에서 학생과 좌석을 선택한다.
3. `고정석 저장`을 누른다.
4. `자리 뽑기`를 실행한다.

### 시스템 반응
- 지정 학생을 랜덤 대상에서 제외한다.
- 지정 좌석에 해당 학생을 먼저 배치한다.
- 나머지 학생만 남은 좌석에 배정한다.

### 완료 조건
- 고정석 학생이 지정 좌석에 유지된 상태로 전체 자리표가 완성된다.

## 유스케이스 4. 저장된 템플릿으로 빠르게 다시 뽑기
### 목표
- 자주 쓰는 반/교실 배치를 다시 구성하지 않고 바로 재사용한다.

### 시작 상태
- 템플릿이 하나 이상 저장되어 있음

### 사용자 행동
1. 상단의 `템플릿` 버튼을 누른다.
2. 원하는 템플릿을 선택해 `불러오기`를 누른다.
3. 필요하면 명단만 조금 수정한다.
4. `자리 뽑기`를 실행한다.

### 시스템 반응
- 명단, 좌석 배치, 통로, 고정석 설정을 현재 초안에 복원한다.
- 결과는 초기화되고 새 배정을 기다리는 상태가 된다.

### 완료 조건
- 사용자는 반복 설정 없이 빠르게 새 자리표를 만든다.

## 유스케이스 5. 결과를 본 뒤 일부 학생만 다시 뽑기
### 목표
- 전체를 다시 섞지 않고 특정 학생만 다시 배정한다.

### 시작 상태
- 최소 1회 이상 자리 뽑기 완료

### 사용자 행동
1. 결과판에서 `일부만 다시 뽑기`를 누른다.
2. 다시 뽑을 학생을 선택한다.
3. `선택한 학생만 다시 뽑기`를 실행한다.

### 시스템 반응
- 선택되지 않은 학생과 고정석은 그대로 유지한다.
- 선택된 학생만 다시 배정한다.

### 완료 조건
- 일부 학생만 바뀐 새 자리표가 생성된다.

## 유스케이스 6. 검색, 복사, 인쇄로 결과 활용하기
### 목표
- 자리표를 바로 확인하고 외부에 전달하거나 출력한다.

### 시작 상태
- 자리 뽑기 결과가 존재

### 사용자 행동
1. 검색창에 학생 이름을 입력한다.
2. 필요하면 `복사` 또는 `인쇄`를 누른다.

### 시스템 반응
- 검색 결과가 있는 좌석만 강조한다.
- 복사는 `행-열: 이름` 형식 텍스트를 클립보드에 저장한다.
- 인쇄는 현재 결과판 중심 레이아웃으로 출력한다.

### 완료 조건
- 특정 학생 좌석을 빠르게 찾고 결과를 공유하거나 출력할 수 있다.

## 유스케이스 7. 이전 결과를 다시 불러오기
### 목표
- 이전 회차의 배정 결과를 참고하거나 다시 활용한다.

### 시작 상태
- 최근 이력이 1건 이상 저장되어 있음

### 사용자 행동
1. 상단의 `이력` 버튼을 누른다.
2. 원하는 회차를 선택해 `불러오기`를 누른다.

### 시스템 반응
- 당시 명단, 좌석 배치, 고정석, 결과를 현재 화면에 복원한다.
- 사용자는 그 상태에서 검색, 복사, 인쇄 또는 새 배정을 진행할 수 있다.

### 완료 조건
- 과거 결과를 현재 화면에서 그대로 확인하거나 후속 작업에 활용할 수 있다.

## UX 우선순위 매핑
- 기본 화면에 반드시 보여야 하는 것
  - 명단 입력
  - 좌석 크기/추천
  - 자리 뽑기 버튼
  - 결과판
- 숨겨야 하는 것
  - 템플릿 목록
  - 이력 목록
  - 좌석 직접 편집
  - 고정석 지정
  - 공정성 옵션
  - 일부 재추첨 대상 선택
- 결과가 나온 뒤에만 보여야 하는 것
  - 전체 다시 뽑기
  - 일부만 다시 뽑기
  - 일부 재추첨 대상 선택
  - 검색 결과 활용

## 핵심 설계 원칙
- 첫 사용자는 `기본 흐름`만 보면 된다.
- 고급 기능은 필요할 때만 펼쳐야 한다.
- 결과판은 항상 가장 중요한 영역이어야 한다.
- 운영 기능은 본문이 아니라 보조 채널에서 열어야 한다.
